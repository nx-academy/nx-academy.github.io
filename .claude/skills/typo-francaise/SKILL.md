# Typographie et registre — français

## Portée et déclenchement

Ce skill ne s'applique qu'au **français**. Avant de l'appliquer :

- Texte en français → appliquer.
- Texte en anglais (SuperJump, Bluesky, tout support EN) → **ne rien appliquer,
  s'arrêter**. Coller une espace avant `:` ou des « … » en anglais serait une
  faute.
- Langue cible ambiguë → demander avant de corriger.

Le skill refuse d'être exhaustif : ce ne sont pas toutes les règles de la
typographie française, mais les quelques-unes qui reviennent réellement dans la
production éditoriale de Thomas. Le reste dilue.

## 1. Ponctuation mécanique

### Signes doubles

Une **espace insécable** précède `;` `:` `!` `?` (en principe une fine insécable
pour `; ! ?` et une insécable pleine pour `:`, mais beaucoup de rendus ne
distinguent pas — rester cohérent, vérifier le rendu final sur le support).

- Correct : « un mot ; puis un autre », « voici : la suite »
- Incorrect : « un mot; puis un autre », « voici: la suite »

Le point `.` et la virgule `,` ne prennent **pas** d'espace avant.

### Guillemets

Guillemets français « … » avec une espace insécable à l'intérieur, jamais les
guillemets droits `"…"` ni les anglais `“…”`.

- Correct : « la sobriété comme signal »
- Incorrect : "la sobriété comme signal"

Guillemets imbriqués : « … “ … ” … » (chevrons à l'extérieur, anglais à
l'intérieur).

### Tiret cadratin

Le tiret cadratin `—` pour les incises et les dialogues, jamais le trait d'union
`-` ni le tiret demi-cadratin `–` à l'anglaise. Espaces autour.

- Correct : « le registre — sobre, observé — est le différenciateur »
- Incorrect : « le registre - sobre, observé - est le différenciateur »
